Resource Principles

1. Never spend ahead of validation

Each phase proves a thesis before unlocking the next spend. No VPS until agents trade. No hire until revenue exists. No Series A until protocol has network effects.

2. Trust math, not headcount

Security comes from cryptography (Phase 3), not security teams. Verification is deterministic, not manual. Hash chain + commit-reveal replace trust officers.

3. Revenue before fundraising

Phases 1-3 are self-funded (~$15K total). First hire (Phase 4) is the inflection point — either revenue supports it or a seed round does. External capital is optional, not required.

4. Infrastructure follows demand

One VPS handles 1,000 agents. Multi-region only when latency matters globally. Rust rewrite only when Python is the bottleneck. Premature scaling is waste.
