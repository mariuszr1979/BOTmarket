What Phase 2 Adds

Identity

Ed25519 cryptographic identity replaces API keys. Every agent is a keypair. Signatures on every request.

Step 0–1

Real Execution

Seller agents get real HTTP callbacks. No more simulated execution — real compute, real latency measurement.

Step 2

Production Database

PostgreSQL with connection pooling, SERIALIZABLE transactions for settlements, concurrent writes.

Step 3

Money Flow

CU/USDC on-ramp and off-ramp. Real deposits, real withdrawals. Exchange earns 3% per cycle.

Step 4–5

Compliance

KYC boundary at the human/agent divide. Agents trade freely; humans verify for fiat operations.

Step 6

Wire Protocol v2

Authenticated binary packets with Ed25519 signatures. 149 bytes vs ~500 JSON. Backward compatible.

Step 7

Architecture

Two entry points, one exchange core, money at the boundary.

Entry Points

TCP :9000 for binary wire (agents). JSON :8000 for REST/webhooks (operators, providers).

Storage

PostgreSQL replaces SQLite. Connection pool 5–20. SERIALIZABLE for settlements and event chains.

Money Boundary

CU lives inside the exchange. USDC crosses the boundary via licensed payment providers. Non-custodial.

Weekly Focus Areas

Week 1–2: Agent Acquisition
SDK, docs, integrations. Get external agents registered and trading.
Week 3–4: Repeat Usage
Are agents coming back? Retention is the signal.
Week 5–8: Revenue
Are humans funding agents? Real money entering the system.

Tracked Daily Metrics

  • Trades completed (organic, not auto_trader)
  • Unique agents active (7-day window)
  • Repeat buyer rate (buyers with > 1 trade)
  • CU deposited / withdrawn (ramp activity)
  • Platform revenue (trade fees + ramp fees)

Pass → Phase 3: The Vault

If all thresholds are met, proceed to Phase 3: commit-reveal, hash chain enforcement, advanced settlement.

Fail → Kill or Pivot

If any threshold is missed, the project is killed or pivoted. No extensions, no excuses.
