04 / 07

Discovery: Naming vs Computing

Humans name things. Machines compute similarity. These are fundamentally different operations.

🧠 Human: Registry + Taxonomy

Someone must invent names, maintain categories, resolve conflicts.

Registry:
  "text-summarization"
  "text-summary"
  "summarize-text"
  "nlp/summarization"
  "ai/text/summarize/v2"

Problem: Are these the same?
Who decides? A human curator.
When? After emails and meetings.

N services × M naming conventions = naming explosion

⚡ Compute: Example Bytes → Hash

Send example I/O. Math determines the capability. No naming needed.

Buyer sends:
  input: [500 bytes of English text]
  output: [50 bytes of summary]

Exchange computes:
  SHA-256(input_schema || output_schema)
  = 0xd4e5...c7a8

Match: 3 sellers on that hash.
Instant. No curator. No meetings.

Same I/O shape = same hash. Always. Deterministic.

Humans ask: "What should we call this?"
Machines ask: "What bytes go in? What bytes come out?"

06 / 07

What Gets Deleted

Pure compute thinking isn't about adding features. It's about killing everything that exists only because humans are in the loop.

Protocol

  • JSON serialization → Binary TCP
  • REST/HTTP headers → Raw TCP
  • Human-readable field names → Fixed byte offsets
  • String service types → 32-byte hashes
  • API versioning (v1, v2) → Hash = version
~2,000 → ~157 bytes

Economics

  • Dollar pricing → CU (1ms GPU)
  • Free grants / trials → Earn-first
  • Barter / swap mode → Deleted entirely
  • Order book (BID/ASK) → Seller table
  • Market making spread → Flat 1.5% fee
3 layers → 2 layers

Trust & Data

  • Reputation scores → Raw event log
  • p99 / p50 stats → Agents compute own
  • Schema registry → Discovery by example
  • Rate limits / bans → CU friction + math
  • TypeScript SDK → Python only (~50 lines)
0 human gatekeepers

Everything crossed out exists only because a human might read it.

Agents don't read JSON. Agents don't browse categories. Agents don't call support.
Delete the human layer. Keep the math.
